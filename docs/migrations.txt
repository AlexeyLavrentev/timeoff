Migrations are applied by the app through Umzug. Core migrations live in the
top-level migrations directory. Edition or premium modules can add their own
migration directories through registry.registerMigrationPath.

To automatically apply any outstanding migrations run following command:

  npm run db-update

Release rule:

  Every deploy/release must execute npm run db-update before the application
  starts serving traffic.

Post-check after migrations:

  1. Verify the command output does not contain errors.
  2. Verify SequelizeMeta contains 20260322120000-add-sso-auth-to-company.js
  3. Verify Companies contains:
     - sso_auth_enabled
     - sso_auth_provider
     - sso_auth_config


How to?

Q: How to add new migration file?
A: Run command like this:

  ./node_modules/.bin/sequelize migration:create --config=config/db.json --models-path=lib/model/db/ --name="add_default_date_format"


Q: How to run pending migrations?
A:

  npm run db-update

Q: How to add premium-only migrations?
A:

  Put the files in the premium module, for example:

    migrations/

  Then register that directory from the module:

    registry.registerMigrationPath(path.join(__dirname, 'migrations'));

  Keep file names stable when moving existing migrations. Applied migration file
  names are stored in SequelizeMeta, so a moved migration with the same file name
  will not run twice.
